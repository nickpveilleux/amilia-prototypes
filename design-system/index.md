# Amilia Design System

This directory codifies Amilia's design system as enforceable rules for AI-driven prototyping. The designer defines constraints and patterns; AI operates within them.

## Reference

- [rules.md](rules.md) — Hard rules (do/don't, constraints). Read before building anything.
- [tokens.md](tokens.md) — Token reference: colors, typography, spacing, shadows, layout.
- [principles.md](principles.md) — Design principles guiding decisions.

## Patterns

- [patterns/index.md](patterns/index.md) — Pattern catalog overview
- [patterns/cards.md](patterns/cards.md) — Card anatomy, variants, usage
- [patterns/tables.md](patterns/tables.md) — Data tables, striping, sorting, pagination
- [patterns/forms.md](patterns/forms.md) — Form layout, validation, field spacing
- [patterns/navigation.md](patterns/navigation.md) — Sidebar, breadcrumbs, tabs
- [patterns/page-layouts.md](patterns/page-layouts.md) — List view, detail view, dashboard
- [patterns/states.md](patterns/states.md) — Empty, loading, error, success patterns

## How to Use

1. Read `rules.md` before any implementation
2. Read the relevant pattern doc when building that pattern
3. Check `tokens.md` when unsure which token to use
4. If a pattern doesn't exist yet, flag it — don't invent conventions
